Introduction

IDI Search App

  • Integrated Data Infrastructure
  • Contains 10's thousands variables across NZ agencies
  • Difficult to know what data is available without having access to it
  • Need to develop and propose research Q to gain access!
  • Need a way to make information about the variables inside the IDI available to anyone
  • Step 1: Generate a list of all variables in the IDI
  • Step 2: Collate that information across multiple DBs into a single, organised, searchable list/database
  • Step 3: Create a simple web app users can use to browse the information
  • A lot of variable names are not very informative
  • Datalab contains Data Dictionaries containing metadata about variables!
  • Step 4: Output data dictionaries from Datalab
  • Step 5: Extract information from data dictionaries and merge with variable lists
  • Step 6: Extend app to make use of this new information

Disclosure Risk Calculator

  • Disclosure risk: identifying an individual in a sample
  • Some simple calculations, and others requiring smcMicro R package
  • A shiny app would be OK, but would either require
  • A. User to manually install/set up (including R?)
  • B. Hosting on server with no security/privacy controls/guarantees (“blackbox”)
  • Preferable solution:
  • 1. Do simple calculations in the browser (almost zero risk of privacy leak)
  • 2. Upload data to disposable R process linked to and only accessible from the current connection
  • Rserve: R package for just this
  • JavaScript client creates a WebSocket connection to R server
  • R server spawns a private R process connected to the WebSocket - inaccessible from anywhere else
  • Once the WebSocket closes, R process killed, along with any uploaded data/memory allocation

Details

  1. User connects to app, R WebSocket connection initiated (private R process)
  2. User uploads data into browser
  3. User chooses variables, sample size, and JS calculates basic stats
  4. Optionally, user can upload stripped data (i.e., label information removed) to a function environment in their R process
  5. R returns a new capability to calcualte summary stats about the uploaded data - data is scoped in the function environment so need not be reuploaded each time
  6. User can modify inputs to see real-time updates
  7. When browser is closed, WebSocket killed, and the R process containing their data completes and dies, releasing the memory (i.e., deleting the data)
